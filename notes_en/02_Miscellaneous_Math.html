              # 2 Miscellaneous Math

              Much of graphics is just translating math directly into code. The cleaner the math, the cleaner the resulting code; so much of this book concentrates on using just the right math for the job. This chapter reviews various tools from high school and college mathematics and is designed to be used more as a reference than as a tutorial. It may appear to be a hodge-podge of topics and indeed it is; each topic is chosen because it is a bit unusual in “standard” math curricula, because it is of central importance in graphics, or because it is not typically treated from a geometric standpoint. In addition to establishing a review of the notation used in the book, the chapter also emphasizes a few points that are sometimes skipped in the standard undergraduate curricula, such as barycentric coordinates on triangles. This chapter is not intended to be a rigorous treatment of the material; instead intuition and geometric interpretation are emphasized. A discussion of linear algebra is deferred until Chapter 5 just before transformation matrices are discussed. Readers are encouraged to skim this chapter to familiarize themselves with the topics covered and to refer back to it as needed. The exercises at the end of the chapter may be useful in determining which topics need a refresher.

              # 2.1 Sets and Mappings

              Mappings, also called functions, are basic to mathematics and programming. Like a function in a program, a mapping in math takes an argument of one type and maps it to (returns) an object of a particular type. In a program we say “type”; in math we would identify the set. When we have an object that is a member of a set, we use the ∈\in∈ symbol. For example,

              a∈S a \in S a∈S

              can be read “aaa is a member of set SSS.” Given any two sets AAA and BBB, we can create a third set by taking the Cartesian product of the two sets, denoted A×BA \times BA×B. This set A×BA \times BA×B is composed of all possible ordered pairs (a,b)(a, b)(a,b) where a∈Aa \in Aa∈A and b∈Bb \in Bb∈B. As a shorthand, we use the notation A2A^2A2 to denote A×AA \times AA×A. We can extend the Cartesian product to create a set of all possible ordered triples from three sets and so on for arbitrarily long ordered tuples from arbitrarily many sets.

              Common sets of interest include:

              • RRR the real numbers;
              • R+R^{+}R+ the nonnegative real numbers (includes zero);
              • R2R^2R2 the ordered pairs in the real 2D plane;
              • RnR^nRn the points in n-dimensional Cartesian space;
              • ZZZ the integers;
              • S2S^2S2 the set of 3D points (points in R3R^3R3) on the unit sphere.

              Note that although S2S^2S2 is composed of points embedded in three-dimensional space, they are on a surface that can be parameterized with two variables, so it can be thought of as a 2D set. Notation for mappings uses the arrow and a colon, for example:

              f:R⟼Z f: R \longmapsto Z f:R⟼Z

              which you can read as “There is a function called fff that takes a real number as input and maps it to an integer.” Here, the set that comes before the arrow is called the domain of the function, and the set on the right-hand side is called the target. Computer programmers might be more comfortable with the following equivalent language: “There is a function called fff which has one real argument and returns an integer.” In other words, the set notation above is equivalent to the common programming notation:

              integerf(real)←equivalent→f:R⟼Z integer f(real) \leftarrow equivalent \rightarrow f: R \longmapsto Z integerf(real)←equivalent→f:R⟼Z

              So the colon-arrow notation can be thought of as a programming syntax. It’s that simple.

              The point f(a)f(a)f(a) is called the image of aaa, and the image of a set AAA (a subset of the domain) is the subset of the target that contains the images of all points in AAA. The image of the whole domain is called the range of the function.

              # 2.1.1 Inverse Mappings

              If we have a function f:A⟼Bf: A \longmapsto Bf:A⟼B, there may exist an inverse function f−1:B⟼Af^{-1}: B \longmapsto Af−1:B⟼A, which is deﬁned by the rule f−1(b)f^{-1}(b)f−1(b) where b=f(a)b = f(a)b=f(a). This deﬁnition only works if every b∈Bb \in Bb∈B is an image of some point under fff (that is, the range equals the target) and if there is only one such point (that is, there is only one a for which f(a)=bf(a) = bf(a)=b). Such mappings or functions are called bijections. A bijection maps every a∈Aa \in Aa∈A to a unique b∈Bb \in Bb∈B, and for every b∈Bb \in Bb∈B, there is exactly one a∈Aa \in Aa∈A such that f(a)=bf(a) = bf(a)=b (Figure 2.1). A bijection between a group of riders and horses indicates that everybody rides a single horse, and every horse is ridden. The two functions would be rider(horse) and horse(rider). These are inverse functions of each other. Functions that are not bijections have no inverse (Figure 2.2).
